## 6. 建议技术方案（待新窗口确认后实施）

### 6.1 渲染层

```
阅读器滚动容器 (reader-scroll / _readerEl)
  ├── 内容层（EPUB HTML / MOBI HTML / TXT / PDF canvas / 漫画 img）  ← 现有
  └── 标注层 AnnotationOverlay（position:absolute; inset:0; pointer-events 按模式切换）
        └── <canvas> 或 SVG（推荐 canvas 2D 笔画 + 形状）
```

- **EPUB/MOBI/TXT：** overlay 挂在 `_readerEl` 同级或内部，随内容滚动；坐标用**内容坐标**（scrollLeft/scrollTop + client 偏移）或**归一化坐标**（相对章节/文档宽高 0~1）持久化。
- **PDF：** 每页独立 overlay（与 `data-page` 容器对齐），标注按 `pageNum` 分组。
- **漫画：** 每页 img 容器上 overlay，按页存储。

### 6.2 状态管理（建议新 store 或 appStore 扩展）

```typescript
// 工具模式
type AnnotationTool = 'none' | 'mark' | 'brush' | 'eraser'

// 画笔配置
interface BrushSettings {
  shape: 'curve' | 'line' | 'polyline' | 'arrow' | 'rect' | 'circle' | 'star' | 'polygon'
  color: string      // hex
  opacity: number    // 0-100
  lineWidth: number  // pt 或 px
}

// 单条标注（持久化）
interface AnnotationStroke {
  id: string
  bookId: string
  format: string
  anchor: { type: 'epub-href' | 'chapter' | 'pdf-page' | 'comic-page' | 'percent'; ... }
  tool: 'brush' | 'mark'
  shape: BrushSettings['shape']
  points: number[][]  // 或 path d 字符串
  color: string
  opacity: number
  lineWidth: number
  createdAt: number
}
```

## 7. 重要决策与约束（仍有效）

1. **EPUB/MOBI 全量渲染**，标注层不得破坏 `_readerEl` / TocPanel `querySelector` 滚动定位。
2. **NCX 用 DOMParser**；MOBI 用 **foliate-js**，不自研。
3. 阅读模式用 **`reader-chrome`** 主题变量，工具面板勿写死 gray 破坏协调。
4. PowerShell 用 `;` 不用 `&&`。
5. 标注不应阻塞启动；按书懒加载 annotations。

## 8. 已知坑（实现时注意）

1. **EPUB `dangerouslySetInnerHTML` 重渲染**会清掉 DOM 内联高亮 → 高亮/标记宜存在 overlay 或 annotation 层，不要直接改章节 innerHTML。
2. **滚动与缩放**：`readerFontSize` 改变后，归一化坐标比绝对像素更稳。
3. **PDF canvas** 与标注 canvas 分离，避免 pdf.js 重绘擦掉标注。
4. **指针事件**：阅读模式默认 `pointer-events:none` 在 overlay 上，仅工具激活时 `auto`，否则无法翻页/选文。
5. **侧栏拖动改宽** 不影响 overlay，但窗口 resize 需重算 canvas 尺寸。
